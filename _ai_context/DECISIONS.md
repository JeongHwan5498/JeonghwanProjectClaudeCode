## [2026-06-16] 로그인 자동화 포기 → 수동 로그인

**결정**: 소셜 로그인은 자동화하지 않고, 사용자가 직접 로그인 후 Enter를 누르면 크롤링 시작

**이유**:
- OAuth 자동화 시 추가 인증(SMS, 캡차 등)이 발생함
- 로그인 자동화에 시간을 쓰는 것은 프로젝트 본질이 아님

**대안이었던 것**:
- storageState.json에 세션 저장 후 재사용 → 세션 만료 시 다시 저장 필요해서 불편
- ID/PW 직접 자동 입력 → 추가 인증으로 막힘

## [2026-06-16] OCR을 스크래핑과 분리

**결정**: `ocr_enabled: False` 기본값. 이미지 저장만 먼저 하고 OCR은 나중에 별도 실행

**이유**:
- 투자 포스팅 이미지(차트, 표, 스크린샷)는 일반 OCR 품질이 낮음
- OCR 실패로 전체 수집이 느려지거나 막히는 것을 방지
- 나중에 GPT/Claude Vision으로 대체할 가능성이 높음

**대안이었던 것**:
- OCR 필수 포함 → 품질/속도 모두 나쁨
- pytesseract만 사용 → 차트/표는 의미 있는 텍스트 못 뽑음

## [2026-06-16] 포스트별 폴더 구조 채택

**결정**: `00_Posts/YYYY-MM-DD_제목/` 구조로 포스트당 폴더 하나

**이유**:
- 이미지, OCR 결과, 메타데이터를 한 폴더에서 관리 가능
- 나중에 Vision 해석 결과도 같은 폴더에 추가하기 쉬움
- 날짜 prefix로 정렬 용이

**대안이었던 것**:
- `post_0001.md` 단일 파일 → 이미지/OCR 관리 어려움
- 전체 images/ 폴더 하나로 통합 → 포스트별 구분 어려움

## [2026-06-16] Playwright Firefox 선택

**결정**: Chromium 대신 Firefox 사용

**이유**:
- Windows 환경에서 Playwright Chromium이 `spawn UNKNOWN` 오류 발생
- Visual C++ 재배포 패키지 설치 후에도 미해결
- Firefox는 동일 환경에서 정상 작동

## [2026-06-16] 포스트 목록 캐시

**결정**: `_post_list.json`에 포스트 목록 저장, 재실행 시 재계산 안 함

**이유**:
- 매번 전체 스크롤하며 목록 수집하면 시간 낭비
- 새 포스트 추가 시엔 파일 삭제 후 재실행으로 갱신

## [2026-06-16] 이미지 파이프라인 — src/data-src/srcset 전부 탐색

**결정**: img 태그의 src뿐 아니라 data-src, srcset도 탐색

**이유**:
- 대상 플랫폼이 lazy loading을 사용할 가능성 있음
- data-src에 실제 URL이 있고 src는 placeholder인 경우 많음
